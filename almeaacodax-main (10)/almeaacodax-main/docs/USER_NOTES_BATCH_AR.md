# ملاحظات الدفعة الجديدة

آخر تحديث: 2026-05-08

هذا الملف هو مكان تجميع ملاحظات المستخدم قبل التنفيذ. الهدف أن ندخل الملاحظات في نفس نهج المشروع: تصنيف، تنفيذ محدود، فحص، ثم إغلاق.

## طريقة التعامل مع أي ملاحظة

1. نكتب الملاحظة كما هي بدون تفسير زائد.
2. نصنفها:
   - خلل يكسر تجربة.
   - تحسين واجهة طالب أو ولي أمر.
   - تحسين لوحة إدارة.
   - بيانات أو نشر أو إنتاج.
3. نحدد شاشة أو ملف أو رحلة واحدة مرتبطة بها.
4. ننفذ أقل تعديل يغلقها.
5. نفحصها بفحص مناسب:
   - متصفح داخلي للواجهة.
   - smoke script إن وجد.
   - `npm run typecheck`
   - `npm run build`
   - `npm --prefix server run check`
   - `npm --prefix server run build`
6. لا نعيد فتح أجزاء مغلقة إلا إذا فشل حارسها أو ظهر خلل واضح.

| الرقم | الملاحظة | التصنيف | المكان المتوقع | الحالة | فحص الإغلاق |
|---:|---|---|---|---|---|

| 2 | التدريب والاختبارات والمكتبة يجب أن تُفتح حسب الباقات: المحتوى المقفول يوجه الطالب للباقات، وبعد شراء الباقة يفتح ما بداخلها. | صلاحيات محتوى + باقات | إدارة التدريب، الاختبارات، المكتبة، صفحة المادة | أغلقت في دفعة الباقات | `npm run build` + `npm run smoke:quiz-access` + فحص المتصفح |

| 4 | الباقات تكون على مستوى المادة أو المسار داخل قسم الباقات في كل مسار. | إدارة باقات | صفحة المسار/الباقات/إدارة الباقات | محفوظة ضمن نظام الباقات الحالي | `npm run smoke:quiz-access` |

## قواعد ثابتة أثناء تنفيذ الملاحظات

- لا نكسر رحلة الطالب الحالية.
- لا نزيد زحمة واجهة الطالب وولي الأمر.
- أي تفاصيل كثيرة تكون في الإدارة.
- لا نغيّر بيانات إنتاج إلا بنسخة احتياطية أو بفحص واضح قبل وبعد.
- لا نكتب أسرار أو مفاتيح في `docs`.
- بعد كل إغلاق نحدث هذا الملف ونقطة المرجع.
