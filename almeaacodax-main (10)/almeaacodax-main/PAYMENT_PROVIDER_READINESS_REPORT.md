# تقرير جاهزية مزودي الدفع

## الهدف

تم تجهيز إعدادات الدفع لتدعم تشغيلًا منظمًا لمصر والسعودية بدون تفعيل بوابة إنتاجية قبل توفير مفاتيح المزود. التعديل محافظ ولا يغير تصميم تجربة الطالب أو الإدارة.

## ما تم تنفيذه

- إضافة حقول مزود الدفع لكل وسيلة:
  - `providerName`
  - `providerCode`
  - `gatewayMode`
  - `supportedCountries`
- حفظ بيانات المزود داخل طلب الدفع:
  - `paymentProviderCode`
  - `paymentGatewayMode`
  - `paymentCountry`
- منع إنشاء طلب دفع لدولة غير مدعومة في إعدادات الوسيلة.
- إضافة اختيارات جاهزة في إدارة المالية:
  - السعودية: Tap Payments, MyFatoorah, HyperPay, STC Pay
  - مصر: Paymob, Fawry, Vodafone Cash
  - التحويل البنكي اليدوي لمصر والسعودية
- عرض اسم المزود والدول المدعومة في نافذة الدفع للطالب بدون تغيير شكل النافذة.

## نمط التشغيل

| النمط | الاستخدام |
| --- | --- |
| `manual_review` | طلب يدوي يراجعه المدير قبل فتح الوصول |
| `payment_link` | المزود يرسل أو يوفر رابط دفع، ثم تتم المراجعة أو الربط لاحقًا |
| `webhook` | اعتماد آلي بعد تحقق توقيع Webhook |

## ما يحتاج مفاتيح خارجية

لا توجد مفاتيح إنتاجية داخل الكود. لتفعيل بوابة دفع حقيقية يلزم إنشاء حساب مزود ثم إضافة القيم في Render/Vercel مثل:

- `PAYMENT_WEBHOOK_SECRET`
- مفاتيح Tap أو MyFatoorah أو HyperPay للسعودية
- مفاتيح Paymob أو Fawry لمصر

## ملاحظات أمان

- واجهة الطالب لا تفتح الوصول بمجرد اختيار وسيلة دفع.
- الاعتماد الآلي عبر Webhook يظل موقوفًا حتى يتم تفعيل `webhookEnabled` وتوفير سر التوقيع.
- طلبات الدفع تحتفظ باسم المزود ونمط التشغيل لتسهيل المراجعة والتقارير لاحقًا.
