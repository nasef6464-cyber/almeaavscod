# تقرير فصل الباقات عن الدورات

## تحديث قفل الفصل من داخل تبويب الدورات - 2026-05-13

- تم حذف كتلة عرض سوق الباقات من داخل تبويب `الدورات` نهائيًا بدل الاكتفاء بشرط إخفاء، حتى لا تظهر الباقات مرة أخرى داخل الدورات بعد أي تعديل لاحق.
- تبويب `الدورات` يعرض الدورات فقط، وزر الدورة المقفولة يفتح طلب شراء من نوع `course`.
- زر `عرض الباقات المناسبة` يوجه إلى صفحة الباقات `tab=packages` فقط، ولا يفتح قائمة باقات عائمة داخل تبويب الدورات.
- تم تحديث اختبار `npm run smoke:package-course-split` حتى يفشل إذا رجعت متغيرات أو كتلة `subjectPublicPackages` داخل تبويب الدورات.

## تحديث توضيح تجربة الطالب داخل تبويب الدورات - 2026-05-13

- تم توضيح تبويب `الدورات` بصريًا ونصيًا حتى لا يظهر للطالب كأنه داخل سوق الباقات.
- كارت الدورة المقفولة يعرض الآن أنها `دورة مدفوعة`، وزرها أصبح `طلب شراء الدورة` ويرسل طلب شراء من نوع `course`.
- تنبيه أعلى تبويب الدورات لا يقول `فتح الباقة` بعد الآن؛ يعرض خيارًا منفصلًا باسم `عرض الباقات المناسبة` لمن يريد باقة تجمع أكثر من جزء.
- شريط `حالة الوصول لهذه المادة` أصبح يكتب للدورات `يحتاج تفعيل` بدل `يحتاج باقة`، حتى لا يختلط مفهوم الدورة المستقلة مع الباقة.
- لم يتم تغيير التصميم أو أماكن العناصر؛ التعديل على النصوص والمنطق فقط حتى يبقى شكل الموقع كما هو.
- تم تحديث اختبار `npm run smoke:package-course-split` حتى يفشل إذا رجعت عبارة `يتطلب باقة أو شراء` أو زر `افتح هذه الدورات الآن` داخل كروت الدورات.

## تحديث إغلاق تبويب الدورات - 2026-05-13

- تم منع عرض سوق الباقات داخل تبويب `الدورات` في صفحة المادة.
- أي رابط قديم من نوع `tab=courses&package=...` يتم تحويله إلى صفحة الباقات بدل فتح نافذة عائمة داخل الدورات.
- أزرار الباقات توجه الآن إلى `tab=packages`، بينما زر الدورة يبقى خاصًا بشراء الدورة نفسها.
- هذا لا يغير التصميم العام ولا شكل البطاقات؛ فقط يمنع الخلط بين منتج الدورة ومنتج الباقة.
- تم توسيع اختبار `npm run smoke:package-course-split` ليمسك هذا الخطأ تحديدًا قبل أي رفع لاحق.

## الهدف

تم تثبيت قاعدة واضحة في المنصة: الدورة منتج مستقل يمكن شراؤه من زر الدورة نفسها، أما الباقة فهي منتج تجميعي يظهر في صفحة الباقات وقد يفتح دورات أو تأسيس أو تدريبات أو اختبارات أو مكتبة.

## ما تم فصله

- صفحة "دوراتي" وصفحات القدرات والتحصيلي وSTEP لا تعرض منتجات الباقات ضمن قوائم الدورات.
- زر الدورة المقفولة في مساحة التعلم يفتح طلب شراء للدورة نفسها، ولا يحول الطالب إلى قائمة باقات عائمة.
- نافذة الدفع التي تعرض اختيارات باقات لا تقبل إلا عناصر تحمل `purchaseType: package` أو `isPackage: true`.
- مسار إنشاء طلب الدفع في السيرفر يرفض إرسال دورة على أنها باقة أو إرسال باقة على أنها دورة.
- معاينة كود الخصم تستخدم نفس حماية الفصل حتى لا يتم اختبار خصم على نوع منتج غير صحيح.

## قاعدة المنتج

| النوع | أين يظهر للطالب | ماذا يفتح |
| --- | --- | --- |
| دورة | صفحة الدورات أو تبويب الدورات داخل المادة | الدورة نفسها فقط |
| باقة | صفحة الباقات الخاصة بالمسار | النطاق المحدد داخل الباقة |
| عضوية عامة | صفحة الباقات إذا كانت مفعلة للنشر | كل النطاقات المحددة في إعدادها |

## طرق الدفع المقترحة لمصر والسعودية

البنية الحالية تدعم طلبات الدفع اليدوية والبطاقة والتحويل والمحفظة، لكن لا توجد مفاتيح إنتاجية مضافة داخل المشروع. التكامل الفعلي يحتاج اختيار مزود وربط مفاتيحه في Render/Vercel.

الخيارات المناسبة:

- السعودية: Tap Payments, MyFatoorah, HyperPay, Moyasar, Stripe إذا كان متاحًا لحسابك.
- مصر: Paymob, Fawry, Accept/Paymob wallets, Meeza عبر مزود محلي.
- التحويل البنكي والمحفظة الإلكترونية يبقيان كمسار يدوي آمن مع مراجعة إدارية.

## ما لم يتم تفعيله بعد

- لم يتم تفعيل بوابة دفع إنتاجية لأن ذلك يحتاج حساب مزود دفع ومفاتيح API.
- لم يتم تغيير تصميم صفحة الباقات أو الدورات.
- لم يتم حذف أي منطق قديم؛ التعديل أضاف حماية وفصلًا فوق الموجود.

## الاختبار

تمت إضافة اختبار عقد:

```bash
npm run smoke:package-course-split
```

هذا الاختبار يثبت أن الدورات لا تدخل قوائم الباقات، وأن السيرفر يرفض خلط `course` و`package`.
