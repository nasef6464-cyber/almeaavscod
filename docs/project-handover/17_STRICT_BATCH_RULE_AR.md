# قاعدة الدفعات الصارمة وحالة الدفعة المفتوحة

آخر تحديث: 2026-05-14

هذا الملف إلزامي لأي حساب Codex أو مطور يستلم المشروع بعد هذه النقطة.

## قاعدة العمل

أي دفعة تطوير يتم الدخول فيها يجب أن تُغلق بالكامل قبل الانتقال إلى دفعة أخرى.

لا يجوز اعتبار أي دفعة منتهية إلا بعد:

- تنفيذ الجزئية end-to-end وليس تحسين جزء صغير فقط.
- فحص الكود المرتبط قبل التعديل وبعده.
- تشغيل الدخان المتخصص للجزئية.
- تشغيل `npm run typecheck` و`npm run build` عند تعديل الواجهة.
- تشغيل `npm --prefix server run build` عند تعديل السيرفر أو العقود المشتركة.
- رفع التغيير إلى GitHub/Vercel إذا كان مطلوبًا ظهوره للمالك.
- فحص الإنتاج على `https://almeaacodax.vercel.app/` وليس المحلي فقط.
- فحص بصري من المتصفح الداخلي لأي تعديل واجهة.
- تحديث ملفات التسليم بما تم وما بقي بصدق.

إذا نقص أي بند، تكتب حالة الدفعة صراحة: `مفتوحة` أو `جزئية` أو `غير مفحوصة إنتاجيًا`. ممنوع وصفها بأنها منتهية.

## حالة آخر دفعة

آخر دفعة تم إغلاقها: **إغلاق فصل الباقات عن الدورات بالكامل**.

Commit الإنتاج: `237317d`

حالة الفحص:

- `npm run smoke:package-course-split`: نجح.
- `npm run typecheck`: نجح.
- `npm run build`: نجح.
- `npm --prefix server run build`: نجح.
- `npm run smoke:frontend`: نجح، وأكد أن Vercel يخدم `237317d`.
- الفحص البصري من المتصفح الداخلي: تم على `tab=courses` و`tab=packages`.
- نتيجة الفحص البصري: تبويب الدورات لا يعرض سوق/كروت الباقات، وصفحة الباقات تظهر فقط عند `tab=packages`.

ما تم تثبيته:

- تبويب الدورات يعرض الدورات فقط.
- كروت الباقات لا تظهر إلا داخل `tab=packages`.
- زر الدورة المقفولة يفتح طلب شراء دورة مستقلة فقط.
- زر عرض الباقات يوجه إلى صفحة الباقات فقط، ولا يفتح نافذة عائمة من داخل تبويب الدورات.
- صفحة الاختبارات المحاكية أو صفحة المسار العامة لا تعرض كروت الباقات إلا إذا كان الرابط `tab=packages`.
- اختبار `npm run smoke:package-course-split` يجب أن يفشل إذا رجع أي خلط بين الباقات والدورات.

أي دفعة تالية يجب أن تبدأ بتحديث هذا الملف باسم الدفعة الجديدة وحالتها، ثم تُغلق بنفس معايير الفحص أعلاه.

## ملاحظة للمالك

تمت إضافة هذه القاعدة لأن بعض الدفعات السابقة وُصفت كأنها مكتملة بينما بقيت بها فجوات. من الآن، التسمية يجب أن تكون دقيقة: منتهية فقط إذا تم تنفيذها وفحصها ورفعها ورؤيتها على الإنتاج.

## تحديث إلزامي جديد (طلب المالك)
- من الآن: عند أمر المالك "اكمل" والانتقال إلى دفعة جديدة، **يُمنع التوقف** حتى تتحقق شروط الإغلاق النهائي للدفعة نفسها.
- تعريف الإغلاق النهائي الإلزامي للدفعة:
  1) تنفيذ كامل نطاق الدفعة end-to-end.
  2) تشغيل فحوص الدفعة الإلزامية وتوثيق نتائجها بوضوح (نجاح/فشل/timeout).
  3) تحديث تقرير الدفعة والـ Ledger وملف الحالة.
  4) تنفيذ **تحقق حي على الإنتاج** (Live Verification) للمسارات/الشاشات المتأثرة.
  5) عدم بدء أي دفعة لاحقة قبل توثيق أن الدفعة الحالية أُغلقت نهائيًا.
- في حال تعذر التحقق الحي بسبب عدم النشر أو انقطاع خارجي: تُسجّل الدفعة "مكتملة جزئيًا" ولا يتم اعتبارها مغلقة نهائيًا.
